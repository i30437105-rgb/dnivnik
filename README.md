# Дневник трейдера — онлайн, бесплатно

Создан 14.07.2026 по требованиям Ивана: сделки тянутся с Bybit по API автоматически,
дневная цель считается от баланса на утро (+3% цель / −2% стоп по канону), статистика
(винрейт, профит-фактор, по монетам, по часам), к каждой сделке — комментарий, состояние
(спокойно/FOMO/отыгрыш...) и скриншот графика.

## Архитектура (всё бесплатно)

- **GitHub Pages** — сам сайт (папка `docs/`, один файл).
- **Supabase (free)** — база данных, вход по паролю, хранилище скриншотов и функция
  синхронизации с Bybit. **API-ключ Bybit хранится только в секретах Supabase** — в браузер
  и на GitHub он не попадает никогда.
- Ключ Bybit нужен **read-only** (только чтение) — торговать и выводить он не может.

## Деплой по шагам (~20 минут, делаем вместе)

**Шаг 1. Supabase.**
1. Зарегистрируйся на supabase.com (можно через GitHub-аккаунт) → New project (регион Frankfurt, пароль БД сохрани).
2. SQL Editor → вставить содержимое `schema.sql` → Run.
3. Authentication → Users → Add user → свой email + пароль (это твой вход в дневник).
4. Authentication → Sign In / Up → отключить «Enable email signups» (чтобы никто чужой не завёл аккаунт).

**Шаг 2. Ключ Bybit (read-only).**
1. Bybit → Профиль → API → Create New Key → System-generated.
2. Права: ТОЛЬКО «Read-Only». Разрешения: Contract — Position, Orders; Account — Wallet. Без Trade, без Withdraw.
3. Сохрани Key и Secret.

**Шаг 3. Функция синхронизации.**
1. В Supabase: Edge Functions → Deploy new function → имя `bybit-sync` → вставить код из
   `supabase/functions/bybit-sync/index.ts` (или через CLI: `supabase functions deploy bybit-sync`).
2. Edge Functions → Secrets → добавить: `BYBIT_KEY`, `BYBIT_SECRET`, `SB_URL` (URL проекта),
   `SB_SERVICE_KEY` (Settings → API → service_role).
3. Автосинк по расписанию: Database → Cron (pg_cron) → задача раз в 30 минут, вызывающая функцию
   (шаблон в дашборде «Invoke edge function on schedule»). Плюс на сайте есть кнопка «Синк» вручную.

**Шаг 4. Сайт на GitHub Pages.**
1. Создай репозиторий (можно приватные данные не бояться — в репо только код без ключей;
   но данные в Supabase, так что репо может быть публичным).
2. Залей папку `docs/`.
3. В `docs/index.html` в самом верху вставь `SUPABASE_URL` и `SUPABASE_ANON` (Settings → API → anon public — этот ключ публичный по дизайну, данные защищены паролем и RLS).
4. Settings → Pages → Source: main branch, папка `/docs` → Save.
5. Через минуту дневник живёт на `https://<логин>.github.io/<репо>/`.

## Что видно на главном экране

- **Карточка дня**: баланс на утро → цель дня в долларах → сколько сделано → сколько осталось →
  стоп-линия −2%. Прогресс-бар. При достижении цели: «рискуем только прибылью дня»;
  при −2%: «торговля до завтра закрыта».
- **Статистика 30 дней**: сделки, винрейт, итог, профит-фактор, средняя прибыль/убыток,
  разбивка по монетам и по часам входа (где зарабатываешь, а где сливаешь).
- **Сделки**: всё из Bybit автоматически; карандаш ✎ — добавить комментарий, состояние, скрин.

## Ограничения v1

- Синхронизируются закрытые позиции фьючерсов (category=linear) за последние 7 дней —
  запускай синк хотя бы раз в неделю (авто-cron это закрывает).
- Баланс начала дня фиксируется первым синком за день: если первый синк случился днём,
  «утренний» баланс восстанавливается как (текущий − сегодняшний P&L) — честная оценка.
- R-мультипликаторы не считаются (Bybit не отдаёт твой стоп) — если захочешь, добавим ручное поле «риск $».
